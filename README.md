# 灵猫符牌（产品原型项目）

一款面向微信小游戏场景的轻量策略卡牌原型。项目围绕“把传统扑克牌玩法改造为有成长、有战斗目标的策略体验”展开，完成了从玩法设计、规则重构、交互反馈到可运行版本的闭环验证。

---

## 项目定位

`灵猫符牌` 是一个用于验证策略玩法方向的产品原型，目标用户为偏轻度但愿意动脑的休闲玩家。  
核心思路是将“单纯比大小/算分”的过程，转化为“构筑 + 出牌 + 伤害结算 + 关卡推进”的战斗式体验，提升每一轮出牌的目标感与决策感。

---

## 项目背景与问题定义

传统扑克牌玩法普遍存在三个问题：

- 规则认知门槛低，但中后期策略空间有限，容易产生重复感。
- 反馈多停留在“得分数字变化”，目标驱动不强。
- 局内成长薄弱，难以形成“这一局和上一局不同”的体验记忆。

围绕这些问题，本项目进行了核心规则重构：

- **移除花色体系**，引入五行元素（金木水火土）。
- **由得分目标改为 Boss 血条目标**，把每次出牌与战斗结果直接绑定。
- **加入成长系统（饰品）**，让玩家在关卡中形成可感知的构筑差异。

---

## 核心玩法设计

### 1) 元素化牌组（替代花色）

- 每关使用五行元素牌组：`金 / 木 / 水 / 火 / 土`。
- 单元素包含 `1-10` 点数。
- 每关重新洗牌并发牌，形成基础随机性与可重复游玩性。

### 2) 牌型驱动的攻击结算

玩家每回合最多选择 5 张牌打出，系统识别牌型并计算伤害：

- 高牌	1张任意牌（无组合）
- 一对	2张点数相同的牌
- 两对	两个不同点数的一对
- 三条	3张点数相同的牌
- 顺子	5张点数连续的牌（元素不限）
- 天火焚原	5张火元素的牌
- 巨浪冲击	5张水元素的牌
- 万木森罗	5张木元素的牌
- 金刚伏魔	5张金元素的牌
- 山崩地裂	5张土元素的牌
- 熔岩流火	3火+2土
- 金水相逢	3金+2水
- 水木清华	3水+2木
- 木火通明	3木+2火
- 土金蕴灵	3土+2金
- 金断朽木	3金+2木
- 木破岩土	3木+2土
- 土掩洪流	3土+2水
- 水灭火势	3水+2火
- 火炼真金	3火+2金
- 葫芦	三条+一对（共5张）
- 四条	4张点数相同的牌
- 同花顺	5张同元素且点数连续的牌
- 皇家同花顺	同元素 10、9、8、7、6（最大顺）

伤害计算逻辑为：

`(牌型基础值 + 有效牌点数 + 饰品加成) x 牌型倍率`

并将结果直接作用于 Boss 血条，形成战斗反馈。

### 3) 资源与操作限制

- 每关限制出牌次数与弃牌次数。
- 玩家需要在“当前收益”和“后续手牌质量”之间做取舍。
- 通过限制机制强化局内决策密度。

### 4) 成长系统（Build 雏形）

当前版本已实现饰品系统（关卡商店购买），效果覆盖：

- 数值增强：如火元素额外加分。
- 倍率增强：如水元素或特定点数提高倍率。
- 触发增强：如首回合倍率翻倍、打出对符额外金币。

饰品机制让每局形成不同“构筑方向”，提升重复游玩的新鲜感。

## 当前版本已落地的产品能力

基于现有代码，已实现的主要功能包括：

- **完整关卡循环**：开始页 -> 对局 -> 结算 -> 商店 -> 下一关 -> 通关/失败。
- **Boss 战斗目标**：以血条替代单纯分数目标，强化目标驱动。
- **牌型识别与结算动画**：从出牌到伤害生效有连续反馈链路。
- **饰品购买与生效**：局内成长可即时影响后续回合策略。
- **信息可视化**：剩余牌库、牌型说明、示例牌组、当前资源状态。
- **基础体验打磨**：音效、动态效果、适配移动端画布布局。

## 简历中强调的完整产品闭环（已完成 + 规划中）

本项目用于展示“从玩法设计到功能落地”的产品能力，闭环包含：

1. **问题洞察**：识别传统扑克玩法单一、反馈弱的问题。  
2. **规则重构**：五行元素化、战斗目标化（Boss 血条）。  
3. **系统设计**：牌型攻击、成长构筑、关卡推进、资源约束。  
4. **原型落地**：基于 Cursor + 微信小游戏框架快速完成可玩版本。  
5. **验证优化**：通过可运行原型持续观察节奏与策略深度，迭代平衡。

  - 在现有饰品（偏数值）基础上增加道具（偏机制），形成更完整 Build 构筑。
- **更细化的 Boss 阶段机制**：
  - 例如阶段技能、抗性变化、特殊回合规则，增强关卡差异性。
- **平衡性与留存导向优化**：
  - 基于测试数据持续优化数值曲线、策略深度与学习成本。
